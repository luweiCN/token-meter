# TokenMeter 设置页「菜单栏外观」卡片设计需求（给设计师 / OpenDesign）

## 产品背景

TokenMeter 设置页（Electron 主界面）现有卡片：Coding Agent 集成 / 供应商额度接入 / 刷新 / 通知 / 数据 / 外观。**本次新增一张「菜单栏外观」卡片**，让用户定制菜单栏常驻组件：切换整体样式 + 控制各元素的显示隐藏。

菜单栏的样式族由另一份稿产出（《菜单栏额度组件样式族设计需求》，5~8 种样式 + 现状同心双环）；本稿中样式画廊的缩略图可先用同心双环占位。布局和交互形式请自由发挥，下面只约定「有什么」和设计系统。

## 设计系统约束（沿用现有设置页，必须一族）

**深色主题（默认）**
- 页面底 `#00111e` · 卡片面 `#02182a` · 内嵌控件面 `#032138`
- 描边/分隔线 `#033259` · 强调色（品牌青）`#0fc5ed`
- 文字：正文 `#cbe0f0` · 标题 `#e6f1fb` · 次要 `#a9b1d6` · 弱化 `#5d84a6`

**浅色主题**
- 页面底 `#f2f7fb` · 卡片面 `#ffffff` · 强调色 `#0895bd`
- 文字：正文 `#0a2540` · 弱化 `#4a6b8a` · 描边 `#d8e6f0`

**组件语言（现有，请沿用其气质）**
- 卡片：surface 底 + 1px 描边 + 8px 圆角 + 16px 内边距
- 卡头：h2 标题（15px 上下）+ 一行 11px 弱化说明
- 开关（sw）、分段选择（seg）、pill 徽标沿用现有形态；数字一律等宽
- 两种主题都要出稿；窗口默认 1180×760，最小宽 720（卡片要能在 720 宽下工作）

**注意**：卡片里出现的「菜单栏组件预览」内容要用菜单栏的色彩语言（系统语义色：绿/黄/红警戒 + 黑白文字），不要把品牌青染进菜单栏预览里——菜单栏里没有品牌色。

## 功能清单（卡片里有什么）

1. **实时预览**：模拟菜单栏效果的预览条，深色、浅色菜单栏底各一条，随下方所有设置即时变化——用户不用抬头看真菜单栏就知道改成了什么样。具体形式可发挥（也可以只画一条、带主题切换）
2. **样式画廊**：每种样式一个缩略预览 + 样式名，单选、选中态要明显；首批约 6~9 种，数量会增长，布局要可扩展（横滑或折行）
3. **元素开关**：品牌名 / 图形 / 数字，三个独立开关
   - 约束：三者至少保一个——只剩最后一个开着时，该开关进入禁用态（禁用态要画出来）
4. **额度窗口**：三选一分段——两个窗口 / 仅 5 小时 / 仅 7 天
5. **今日用量**：三选一分段——今日 token / 今日花费 / 隐藏
6. **按供应商显示**：每家供应商一行（名称 + 「显示在菜单栏」开关），当前 3 家（Claude Code / Codex / 智谱 GLM），会增长
   - 注意：这个开关**独立于**「供应商额度接入」卡里的整体启停（那个会停掉数据刷新，这个只影响菜单栏显不显示）。行内要有一句轻量说明，别让用户混淆两者
7. 所有改动**即时保存**（沿用现有「已保存 ✓」角标 + 失败 toast 机制），没有保存按钮

## 交互与状态要求

- 加载态、保存失败的内联提示都要覆盖
- 菜单栏应用未运行时：预览照常工作（纯前端渲染），加一行弱化说明「菜单栏应用未运行，改动将在其下次启动时生效」
- 720 宽度下布局仍要成立（画廊可折行/横滑）
