# TokenMeter 基础表单控件设计需求(给设计 Agent 的提示词)

你之前为 TokenMeter 设计了整套主界面(coolnight 主题的设计系统:总览、项目、会话、设置等页面)。现在应用引入了一批**表单/筛选交互控件**,当前用的是组件库默认样式,与你的设计系统完全不统一(视觉上是两种语言)。请按你的设计系统为下列控件出一套完整设计。

## 交付格式

**一个自包含的 HTML 文件**(内联 CSS,不依赖外部资源),按组件分节展示:

- 每个组件的**全部状态**:默认 / hover / focus(键盘可见焦点)/ active / disabled / 选中 / 错误(如适用);
- **深色为主 + 浅色主题**(现有系统 `:root[data-theme="light"]` 机制,两套都要);
- CSS 请引用现有设计 token 变量名(`--bg / --surface / --surface2 / --fg / --fg2 / --muted / --border / --border-strong / --accent / --on-accent / --danger / --warn / --ok / --focus / --track` 等),我方直接移植规则;
- 浮层类组件把「触发器 + 打开态浮层」并排画出来。

## 组件清单

### A. 按钮 Button

- 变体:主操作(accent 实底)/ 次要(描边)/ 幽灵(无底,hover 显)/ 危险(danger);
- 尺寸:sm(筛选行,高度与筛选控件一致)/ md(常规);
- 形态:纯文字、图标+文字、纯图标(方形);
- 特别注意:**loading 态**(转圈+文字)不得改变按钮尺寸——历史教训:刷新按钮 loading 时高度差 1px 引发整页抖动。

### B. 文本输入 Input / 搜索框 SearchInput

- 普通文本输入;
- 搜索框:放大镜前缀图标 + placeholder + 内容非空时尾部小清除图标(✕,现有 ClearIcon 交互);
- 焦点态、禁用态、错误态。

### C. 时刻输入 TimeField(重点,当前最丑)

- `HH:mm` 输入(时间点筛选的核心控件),当前是原生 `input[type=time]` 的 `--:--` 系统样式;
- 需要与输入框同语言的定制外观;可以是「两段数字 + 冒号」的分段编辑,或带掩码的单输入框——交互形态由你定;
- 有禁用态(未选日期时)。

### D. 单选下拉 Select

- 触发器:当前值 + chevron;浮层:选项列(选中项带勾/高亮);
- 用于排序选择(「按 Token / 按成本 / 按时间」)等,选项数 3-8 个;
- 键盘导航(上下键 + 回车)的视觉反馈。

### E. 多选下拉 MultiSelect

- 触发器文案:「全部项目」/「N 项已选」;
- 浮层:顶部搜索框 + 复选选项列(勾选态)+ 底部「清除」;
- 用于项目筛选(选项可能几十个,浮层列表可滚动)。

### F. 日期时间范围选择器 DateTimeRangePicker(重点,当前整体最烂)

这是新的核心筛选控件,场景:「从额度刷新的那一分钟起,统计到现在/周期末的用量」。

- **触发器**:日历小图标 + 范围文案(`7/14 14:32 – 7/20 23:59`,未选时「全部时间」)+ 有值时尾部清除小图标;
- **浮层**布局:左侧快捷项竖列(今天 / 本周至今 / 本月至今 / 近 7 天 / 全部时间)+ 右侧日历 + 日历下方「起 / 止」两个时刻输入(用 C 的 TimeField);
- **日历**:月标题 + 前后月导航;范围高亮(起点/终点圆角端、中段连续底色)、今天标记、非本月日期弱化、hover 到第二个日期时的**范围预览高亮**(这些交互在你之前给会话页设计的 DateRangePicker 里都有,可沿用视觉语言);
- 半开区间形态:只选了起点时,文案显示「起点 – 现在」。

### G. 开关 Switch

- 设置页数据源/供应商启停(现有 src-toggle 附近);开/关/禁用三态。

### H. 滑条 Slider

- 设置页通知阈值(50%–100%);轨道 + 滑块 + 当前值标注。

### I. 分页器 Pager

- 页码 + 上一页/下一页;当前页高亮;省略号折叠(页数多时)。

## 已有良好实现、无需重新设计(仅供风格锚定参照)

- 统计卡 StatTile、图表 tooltip(.tip)、抽屉 Drawer、热力图、趋势图图例;
- 表格(sess-table:排序表头、行 hover)。

## 现状问题示例(为什么要重做)

- 日历弹层曾渲染成系统灰按钮(已修掉最烂的层),但目前仍是组件库默认的黑白灰,与 coolnight 的 accent 青色语言不符;
- 时刻输入是原生 `--:--` 系统控件;
- 快捷项按钮、日期格子的尺寸/密度/圆角与设计系统的紧凑风格不一致。
