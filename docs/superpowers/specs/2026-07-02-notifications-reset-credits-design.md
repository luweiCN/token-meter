# TokenMeter 通知与 Codex 重置卡设计

## 背景

TokenMeter 当前已经能展示 Codex、Claude Code、智谱等供应商的额度，并且会把最近一次可用数据缓存到本地。下一步要补上两类能力：

1. 展示 Codex 重置卡数量，并在用户点开后展示每张卡的发放时间和过期时间。
2. 对关键额度变化发送 macOS 通知，同时避免重复通知。

本设计只覆盖 macOS 第一版，不引入跨平台通知抽象。

## 数据来源

Codex 重置卡使用本机 Codex 登录凭证请求：

`https://chatgpt.com/backend-api/wham/rate-limit-reset-credits`

认证方式：

- 读取 `~/.codex/auth.json`。
- 使用 `tokens.access_token` 作为 Bearer token。
- 不记录、不展示 access token、refresh token、cookie 或完整唯一 ID。

接口已验证会返回 `credits` 明细数组。当前本机查询结果为 6 张卡，每张卡都包含可转换为北京时间的发放时间和过期时间。

如果接口返回 401，TokenMeter 视为 Codex 凭证失效或 Authorization header 未正确携带，并在 UI 中展示为 Codex 重置卡读取失败，不触发重置卡相关通知。

## 数据模型

在核心模型里给 `ProviderUsageSnapshot` 增加可选的扩展权益信息：

- `resetCredits`：只在供应商支持重置卡时存在。
- `availableCount`：当前可用重置卡数量。
- `credits`：重置卡列表。
- 每张卡只保存 UI 必需字段：
  - 发放时间。
  - 过期时间。

不把完整唯一 ID 放入模型，也不写入缓存。后续如果需要精确识别每张卡，最多保存服务端 ID 的不可逆摘要，但第一版不做。

旧缓存没有 `resetCredits` 字段时应能正常解码，默认视为没有重置卡数据。

## UI 设计

Codex 供应商卡片默认保持紧凑，只增加一行重置卡摘要：

`重置卡 6 张`

这一行默认不展开。用户点击后展开详情区，显示每张卡：

- `发放：2026-06-18 08:32`
- `过期：2026-07-18 08:32`

详情区作为 Codex 卡片里的次级信息处理，视觉层级低于主额度和附加额度，避免抢占额度展示的主层级。

如果接口失败但有旧缓存，继续显示旧的重置卡数据，并在状态提示里说明数据没有及时更新。

## 通知规则

通知由一次刷新前后的快照差异触发。首次启动、首次加载缓存、刷新失败、只读到缓存数据时不触发通知。

### 新重置卡

当 Codex 上一次成功快照的 `availableCount` 小于这一次成功快照的 `availableCount` 时，发送一次通知：

`Codex 新增重置卡：+N，当前共 M 张`

如果应用重启，缓存里的上一次快照会作为比较基准，因此同一次新增不会反复通知。

### 额度刷新

对每个额度项独立判断。当前一次剩余百分比低于 95%，这一次剩余百分比达到 99% 或以上，并且该额度项状态正常时，发送一次通知：

`Codex 5 小时额度已刷新`

如果额度项有 `resetAt`，刷新后 `resetAt` 应该晚于之前的 `resetAt`；没有 `resetAt` 时只使用剩余百分比跳变判断。

### 额度归零

对每个额度项独立判断。当前一次剩余百分比大于 0，这一次小于等于 0 时，发送一次通知：

`Codex 5 小时额度已用尽`

因为通知只在跨越边界时触发，所以停留在 0 不会重复通知。额度恢复后再次归零，才会再次通知。

## 去重策略

第一版不额外维护复杂通知状态表，而是利用已有快照缓存做边界判断：

- 上一次快照来自内存或磁盘缓存。
- 这一次快照来自成功刷新结果。
- 只有状态从 A 跨到 B 时触发通知。

这样可以自然避免以下重复场景：

- 应用启动时看到已有 6 张重置卡。
- 连续刷新时额度一直是 100%。
- 连续刷新时额度一直是 0。
- 接口失败时复用缓存数据。

如果后续发现某个接口会返回抖动数据，再增加单独的通知状态文件。

## 组件划分

### TokenMeterCore

- 解析 Codex 重置卡接口响应。
- 扩展 `ProviderUsageSnapshot` 模型。
- 提供纯函数通知事件检测器，输入旧快照和新快照，输出通知事件。
- 为解析和通知边界判断补测试。

### TokenMeterApp

- 新增 Codex 重置卡请求逻辑。
- 在刷新成功后调用通知事件检测器。
- 使用 `UserNotifications` 发送 macOS 通知。
- 在 App 启动时请求通知权限。
- 在 Popover 里显示 Codex 重置卡摘要和可展开详情。

## 错误处理

- 401：提示 Codex 凭证失效或 Authorization header 无效。
- 网络错误：提示重置卡接口请求失败。
- JSON 解析失败：提示接口返回格式异常。
- 所有错误都不能打印 token、cookie、完整 ID。
- 错误时保留旧缓存，避免 UI 从正常数据跳成空数据。

## 测试

需要补充以下测试：

- Codex 重置卡响应解析出数量、发放时间、过期时间。
- 旧缓存缺少 `resetCredits` 时能正常解码。
- 新增重置卡只触发一次事件。
- 额度从低值跳到 100% 触发刷新事件。
- 额度停留在 100% 不重复触发刷新事件。
- 额度从大于 0 到 0 触发归零事件。
- 额度停留在 0 不重复触发归零事件。
- 刷新失败或状态异常不触发通知事件。

实现完成后运行 `swift test` 验证。

## 范围外

- 不做 Windows、Linux 通知。
- 不做通知偏好设置页。
- 不做声音、勿扰时间段、供应商级通知开关。
- 不做 OpenCode Go 的额度查询。
