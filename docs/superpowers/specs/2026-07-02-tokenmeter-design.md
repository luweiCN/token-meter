# TokenMeter 设计文档

## 产品定位

TokenMeter 是一个 macOS 菜单栏小工具，用来查看 AI 供应商的 token 用量、额度、余额或费用相关信息。

第一版先保持很小的范围：

- 作为轻量级 macOS 菜单栏工具运行。
- 在菜单栏里显示指定供应商的用量或剩余额度。
- 点击菜单栏后，用原生浮窗展示更详细的信息。
- 通过本地配置文件读取供应商配置。
- MVP 阶段不做复杂的供应商管理和设置页面。

这个产品应该像一个安静的小工具，而不是一个很重的管理后台。空闲状态下的内存占用是核心产品要求。

## 命名

- 产品名：`TokenMeter`
- 文件夹 / 仓库名：`token-meter`
- App 显示名：`TokenMeter`
- Swift 模块名：`TokenMeter`
- Bundle ID：`com.luwei.tokenmeter`

这个名字比只强调 quota 的名字更宽一些。后续它可以覆盖 token 用量、费用、历史趋势、本地 coding agent session 统计和供应商管理。

## 平台范围

第一版只做 macOS。

Windows 和 Linux 暂时不进入 MVP 范围。Electron、Tauri 这类跨平台方案暂时不作为第一版首选，因为这个产品最重要的是常驻时足够轻，而不是一开始覆盖更多平台。

## 长期架构

我们认可的长期方向是一个混合架构的 macOS App：

```text
原生 macOS 常驻层
├─ 菜单栏入口
├─ 快速浮窗
├─ 后台刷新
├─ 供应商 API 客户端
├─ Keychain 密钥存储
├─ 本地缓存 / SQLite
└─ 本地文件扫描

按需打开的 Web 控制台
├─ 供应商管理
├─ 用量图表
├─ 历史分析
├─ 本地 coding agent session 可视化
└─ 高级配置
```

核心约束是：常驻层必须是原生的。菜单栏展示、后台刷新、供应商请求、本地扫描和密钥处理都不能依赖 WebView。

Web UI 只在后续复杂页面需要时按需打开，比如供应商管理、图表、历史分析、本地 session 可视化等。

## MVP 架构

MVP 要比长期架构更简单：

```text
Swift / AppKit 常驻应用
├─ NSStatusItem 菜单栏显示
├─ 原生 SwiftUI / AppKit 浮窗
├─ 本地配置文件读取
├─ 供应商用量查询
├─ 轻量本地缓存
└─ 需要真实密钥时接入 Keychain
```

MVP 不需要 WKWebView。等后续供应商管理、图表或者本地 session 可视化变复杂之后，再引入 Web 控制台。

## 技术选择

MVP 使用 macOS 原生技术：

- 用 AppKit 做菜单栏集成。
- 在适合的地方使用 SwiftUI，提高浮窗或简单设置 UI 的开发效率。
- 用 Swift Concurrency 和 `URLSession` 请求供应商 API。
- 用 Keychain 存储密钥。
- 用简单本地文件作为第一版供应商配置。
- 如果后续需要记录历史用量，再引入 SQLite。

不使用 Electron 作为第一版技术方案，因为它对一个常驻菜单栏小工具来说太重。

不使用 Tauri 作为第一版 App 壳。只有当未来跨平台优先级超过 macOS 低内存体验时，才重新考虑 Tauri。

## 配置方式

MVP 先用本地配置文件，不做完整的供应商管理 UI。

配置文件需要描述：

- 启用哪些供应商。
- 展示名称。
- 供应商类型。
- 凭据引用。
- 菜单栏展示偏好。
- 刷新间隔或默认刷新策略。

一旦支持真实凭据，密钥不要直接明文写在配置文件里。配置文件可以引用存放在 Keychain 里的凭据。

## 用户体验

MVP 不做主窗口。

预期交互：

1. 用户启动 TokenMeter。
2. TokenMeter 出现在 macOS 菜单栏。
3. 菜单栏展示一个紧凑的用量或额度信号。
4. 用户点击菜单栏项，打开原生浮窗查看供应商详情。
5. App 按保守的刷新频率在后台更新数据。

界面要紧凑、信息密度高，不做营销页或复杂仪表盘。除非用户后续主动打开分析控制台，否则日常使用应该只停留在菜单栏和小浮窗里。

## 隐私与安全

TokenMeter 应该坚持本地优先。

- 供应商凭据只保存在用户本机。
- 凭据优先使用 Keychain。
- 供应商用量数据缓存在本地。
- MVP 不做云同步。
- 未来如果做本地 coding agent session 扫描，必须让用户明确触发，并清楚选择要扫描的目录。

## 暂缓功能

以下功能不进入 MVP：

- 复杂供应商管理 UI。
- 用量图表。
- 历史分析。
- 本地 coding agent session 扫描。
- WKWebView 控制台。
- Windows 或 Linux 支持。
- 云同步。
- 账号系统。
