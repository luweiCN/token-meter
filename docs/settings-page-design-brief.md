# TokenMeter 设置页设计需求(给设计师 / OpenDesign)

## 产品背景

TokenMeter 是一个 macOS 桌面应用(菜单栏常驻 + Electron 主界面),监控本地 coding agent(Claude Code、Codex CLI、OMP、OpenCode)的 token 用量、花费与订阅额度。主界面已有四页:总览 / 会话 / 索引状态 / 设置。**本次要设计的是「设置」整页**,请与总览页的既有视觉语言保持一族(总览页截图随附)。

## 设计系统约束(必须沿用)

**深色主题(默认)**
- 页面底 `#00111e` · 卡片面 `#02182a` · 内嵌控件面 `#032138`
- 描边/分隔线 `#033259` · 强调色(品牌青)`#0fc5ed`
- 文字:正文 `#cbe0f0` · 标题 `#e6f1fb` · 次要 `#a9b1d6` · 弱化 `#5d84a6`

**浅色主题**
- 页面底 `#f2f7fb` · 卡片面 `#ffffff` · 强调色 `#0895bd`
- 文字:正文 `#0a2540` · 弱化 `#4a6b8a` · 描边 `#d8e6f0`

**组件语言(现有,请沿用其气质)**
- 卡片:surface 底 + 1px 描边 + 8px 圆角 + 16px 内边距
- 卡头:h2 标题(15px 上下)+ 一行 11px 弱化说明文字
- 页头:h1 + 副标题小字;数字一律等宽字体
- 两种主题都要出稿;窗口默认 1180×760,最小宽 720(设置页要能在 720 宽下工作)

## 页面功能清单(信息架构)

### A. Coding Agent 集成(核心模块)
- **只列出本机检测到的 agent**(检测依据:安装目录/命令存在);一个都没检测到时要有空态
- 每行:agent 名称、集成方式简述、**集成开关**
  - 开 = 安装状态上报 hooks/插件 + 该 agent 计入统计
  - 关 = 卸载 hooks/插件 + 从所有统计维度排除
- 集成状态徽标:已集成 / 未集成 / 集成失败;开关切换后要提示「从该 agent 的下一个会话开始生效」
- 当前支持:Claude Code、Codex CLI、OMP、OpenCode;列表会增长,设计要可扩展

### B. 供应商额度接入
- **Codex、Claude Code:自动读本机登录凭证**,只显示状态(已连接/读取失败),无需用户配置
- **智谱(及将来需要 API Key 的供应商):双轨凭证**
  - 应用内填写(优先,存系统钥匙串)——需要一个凭证输入交互(脱敏显示、可清除)
  - 环境变量回退(自动检测,显示「从环境变量读取」状态)
  - 状态:已连接 / 未配置 / 连接失败(带简短错误)
- 每个供应商:启用开关、自定义显示名、是否显示在菜单栏弹窗/图表
- **注意:不再有「菜单栏主供应商」选择项**(菜单栏已改为显示今日 token 总量)

### C. 刷新
- 本地会话扫描间隔(默认 60 秒,可调)
- 额度刷新频率说明(固定至少 5 分钟,防上游接口限流——只读说明即可)

### D. 通知
- 额度告警:开关 + 阈值
- 系统通知授权状态展示 + 跳转系统设置的入口

### E. 数据
- 扫描根目录列表(启用/禁用/添加自定义路径)
- 重建索引入口(危险操作样式,需确认)
- 数据库位置展示(可点击在访达中显示)

## 交互与状态要求
- 所有开关/输入即时保存;保存失败要有内联错误提示(现有乐观锁机制,冲突时提示重试)
- 加载态、空态、错误态都要覆盖
- 分区之间的视觉层级:A 和 B 是高频核心,C/D/E 次之(可考虑折叠或次级分组)
